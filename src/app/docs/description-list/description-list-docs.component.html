Use description list when users need to view a set of simple data with labels. Use description list to make the data easy to scan and to make the relationships to the respective labels easy to understand.

Use description list with unordered lists when multiple values are associated with the same terms.

Use description list in the "long description" mode when users need to view a set of terms with long text descriptions.

Don't use description list when the information is important and should be emphasized in the information hierarchy. Use key info or other text styles instead.

Don't use description list when information can be understood on its own within its context. Use standalone text without a term or label.

Don't use description list with unordered lists for multiple values in a context where unpredictable heights can harm the layout.

Term Description

Description lists in a horizontal layout will respond to the width of their container by changing to a 2-column layout and then a 1-column layout when appropriate.

Description lists in long description mode will respond to the width of their container by wrapping text first and then stacking the term above the decsription when appropriate.

Don't use colons or other punctuation after description list terms.

Use horizontal layout for a list of up to 6 terms when it stands alone in a composition.

Use vertical layout to enable easier scanning when the description list is combined with other content in a composition, or when a container holds multiple lists. Use fluid grid to arrange multiple description lists inside a container.

Use descriptive headings with multiple lists to make scanning content easier.
